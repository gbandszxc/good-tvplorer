# 设计规范

## 元数据

- 产品：Good TVplorer
- 类型：product
- 平台：Android TV
- UI 技术栈：Jetpack Compose + Material 3 基础组件

## 设计意图

Good TVplorer 是客厅电视上的媒体文件工具。方向采用“冷静的 NAS 媒体库”：比传统文件管理器更适合图片、音频、视频的快速预览，但仍保持工具界面的密度、速度和遥控器优先。

## 色彩

当前配色来自 `TvTheme.kt`，整体应保持克制：以冷静的石板蓝承载媒体库界面，用琥珀色表达焦点。

| 角色 | Hex | 用途 |
| --- | --- | --- |
| 背景 | `#0B121A` | App 背景和预览区域底色 |
| 导航栏 / 面板 | `#101A26` | 左侧 Dock、首页 SMB 面板、快速预览面板 |
| 表面 | `#152232` | 按钮、列表行、网格卡片 |
| 主色 | `#FFC857` | 焦点表面和高优先级操作反馈 |
| 主色文字 | `#151007` | 焦点主色上的文字和图标 |
| 背景文字 | `#F3F7FA` | 主要文字 |
| 次要色 | `#7CC7D8` | 区块标题、低调状态强调 |
| 错误 | `#FFA3A3` | 错误消息 |
| 弱化文字 | `#A8B8C7` / `#728397` | 次级说明和遥控器提示 |
| 设置侧栏 | `#0E1824` | 设置 Activity 的一级菜单背景，用于与右侧内容区分层 |
| 设置分隔线 | `#26384B` | 设置侧栏与内容区的 `1dp` 边界，以及非焦点设置卡片描边 |

主色用于焦点和确认反馈，不作为装饰色。MVP 阶段避免 Material You 风格的动态配色；电视端优先保证可读性和一致性。

## 字体

使用 Compose 默认的单一无衬线字体族。产品 UI 使用固定字号，不使用流式字体。

运行时字体缩放中的 `100%` 表示 Good TVplorer 的应用标准字号，默认值为 `100%`。当前内部使用 `0.75` 校准系数确定电视端的标准视觉大小；该系数不等同于 Android 原始字体比例，也不作为用户可见百分比。设置页面提供 `80%-120%` 九档，步长为 `5%`；新增文字必须在全部档位下保持可读。

- App 标题：`40sp`
- 页面路径 / 标题：`26-32sp`
- 列表项标题：`22sp`，semibold
- 列表元信息：`16sp`
- 预览正文：`24sp`，`34sp` 行高
- 按钮文字：`24sp`，semibold
- 网格卡片标题：`18sp`，semibold
- 设置页页面标题：`32sp`
- 设置行标题：`22sp`
- 设置行说明 / 尾部值：`18sp`
- 设置页控制文字：`20sp`
- 设置页缩放百分比：`28sp`

文字必须完整容纳在焦点表面内。长文件名在列表行中单行截断；在空间允许的预览页中展示完整名称。

## 布局

主界面使用固定 Dock 布局，不使用卡片套卡片。启动后默认显示本机根目录；顶部来源导航在本机与网络入口间切换，并与内容区共用背景，保持约 `46dp` 紧凑高度。左侧 Dock 宽 `60dp`，浏览页的视图切换、刷新和返回以 SVG 图标从上向下排列；连接管理（Link 图标）和设置图标固定在底部。

```text
Browser
+----------------------------------------------------------------+
| 网格 |                     本机    [ 网络 ]                    |
| 刷新 | [排序] [编辑] /当前路径                 [搜索当前目录]   |
| 返回 |  file grid/list main surface              | 快速预览      |
|      |  file grid/list main surface              | 快速预览      |
| 连接 |  [cover] [cover] [cover]                  | cover/thumb   |
| 设置 |  [cover] [cover] [cover]                  | file details  |
+----------------------------------------------------------------+
```

间距需要适合电视端观看：主内容区相对左侧 Dock 和屏幕右边界统一留 `24dp`；屏幕上边界到来源导航、来源导航到地址栏、地址栏到文件区与预览区、文件区与预览区到底边界均统一留 `16dp`，文件区与预览区间留 `16dp`。列表行内边距约 `18dp`，行间距约 `10dp`，可聚焦目标高度必须足够远距离选中。

顶部来源导航仅显示“本机”和“网络”，不追加当前状态文字。当前来源以文字周围的深色圆角矩形和柔和阴影表示；遥控器焦点仍使用琥珀表面与浅色描边，确保远距离可辨。来源切换项高度为 `48dp`，兼顾遥控器聚焦和触屏点击。

左侧 Dock 的图标按钮为 `48dp`，使用 `12dp` 圆角；焦点态使用琥珀表面和 `2dp` 浅琥珀描边，图标保持 `24dp`，在窄栏中仍清晰可辨。

浏览页不显示产品标题。顶部来源导航与工具栏、工具栏与内容区均使用 `16dp` 间距；工具栏统一为 `48dp` 高的三段式：左侧 `48dp` 排序 SVG 按钮，中间路径栏，右侧 `240dp` 搜索框。三段间距为 `12dp`，路径栏与搜索框均使用导航栏色圆角矩形容器。路径栏默认仅左对齐回显当前路径；选择编辑图标后切换为同高度的可输入地址栏，提交后跳转到输入路径，Back 优先退出编辑态。排序按钮打开可遥控器聚焦的菜单，每项以文件名、文件大小或时间 SVG 标识，并用 `↑` / `↓` 表示正序/倒序。内容区首项固定为“返回上一级”：列表为置顶行，网格为左上卡片；空目录时该项自动获取焦点。搜索框输入后即时检索当前目录树（含子目录）中的名称匹配项；结果刷新不得移动焦点，焦点保持在搜索框，直到用户自行使用方向键移开。由搜索框按下键进入内容区时，按空间关系进入结果项，不恢复目录焦点锚点。Back 在有搜索词时优先清空搜索词；目录始终排列在文件之前。

## 组件

### TvButton

- 形状：`8dp` 圆角
- 默认文字为 `20sp`，内边距为水平 `20dp`、垂直 `10dp`；文字继续响应显示设置中的字体缩放。
- 默认态：表面色背景、浅色文字
- 焦点态：主色背景、深色文字、`3dp` 浅色描边
- 行为：可聚焦、可点击、可用方向键确认
- 触控：可点击区域高度不小于 `48dp`

### 文件列表行

- 形状：`8dp` 圆角
- 默认态：表面色背景
- 焦点态：琥珀主色背景、深色文字、`3dp` 浅琥珀描边
- 紧凑尺寸：缩略图 `48dp`、行内边距 `8dp`、行间距 `8dp`；默认视口目标可显示约 6 行。
- 内容：缩略图或类型标记、文件名、大小 / 时间元信息
- 排序：目录行排在文件之前

### 文件网格卡片

- 用于网格模式下浏览图片、音频和视频。
- 缩略图比例：`16:10`。
- 默认固定为每行 4 项，卡片内边距 `8dp`、卡片间距 `10dp`；文件名单行截断，完整名称由快速预览展示。
- 包含“返回上一级”在内的所有网格卡片均使用相同的 `16:10` 媒体区和固定高度标题区，文件名再长也不得改变卡片或网格行尺寸。
- 非焦点状态的长文件名显示省略号；获得遥控器焦点后自动单行横向滚动，以便查看完整名称。
- 图片缩略图只为当前 LazyGrid / LazyColumn 可见项请求；离屏、切换目录或打开原图时取消未完成请求。
- 缩略图加载期间使用现有 `IMG` 类型标记占位，完成后原位替换，不改变卡片尺寸和焦点。
- 音频缩略图优先显示内嵌封面，缺失时显示唱片 SVG；视频缩略图取约第 3 秒的关键帧；文本显示文档 SVG。
- 焦点态与列表行使用相同的琥珀焦点表面。

### 快速预览面板

- 浏览页右侧固定面板。
- 展示当前焦点项的缩略图 / 封面 / 帧图、文件名、类型、大小和时间；图片补充分辨率与格式，音频补充时长、码率、标题、艺术家、专辑、流派、年份和音轨等可用标签，视频补充时长、码率和格式。元数据读取在后台进行，焦点切换时取消过期任务，不阻塞浏览操作。SMB 快速预览只读取图片头部及媒体文件头/尾各最多 `512 KiB` 的必要字节；无法快速识别的格式省略详细字段，不为元数据下载完整文件。
- SMB 图片快速预览优先从最多 `256 KiB` JPEG 前缀提取 EXIF 内嵌缩略图；没有有效内嵌图时，从 SMB 流按采样倍率解码并只保存最终缩略图。
- 打开 SMB 原图时先显示已有缩略图，再由 Coil 直接读取 SMB 流；不缓存原图文件。
- SMB 音频封面与视频帧通过范围读取供系统媒体解析器按需提取，不完整缓存媒体；文本快速预览显示头部摘录。

### 图片查看页

- 在 `MainActivity` 内呈现，不另启 Activity。默认隐藏状态栏和导航栏，原图不留额外内边距，以适应屏幕模式顶天立地显示；遥控器或边缘手势可短暂唤出系统栏。按左右键按当前目录排序切换上一张或下一张。
- 触屏单击图片显示或隐藏顶部控制栏与底部胶卷；左右滑动按当前目录排序切换图片。胶卷缩略图可直接点击切换。
- 按确认键显示控制层：顶部为完整文件名和返回按钮，底部为按相同排序排列的横向图片胶卷。胶卷展开时自动定位当前图片并获取焦点，胶卷项可立即用左右键选择并按确认切图。
- 控制层显示时，遥控器 Back 仅关闭控制层；仅在控制层隐藏时 Back 返回文件列表。顶部“返回”按钮始终直接返回文件列表。
- 胶卷使用 LazyRow，仅为可见缩略图请求加载；离屏或退出查看页时取消未完成请求。胶卷内切换当前图片不得取消其他仍可见项的缩略图请求。

### 音频与视频播放页

- 音频左侧显示内嵌封面或 `AUD` 占位，右侧显示同目录同名 `.lrc` 歌词；视频主体显示画面，并在底部叠加 Media3 解码的内嵌字幕或同目录同名 `.srt` 字幕。
- 视频默认沉浸全屏播放，确认键显示顶部文件名/返回按钮和底部控制层；Back 优先收起控制层，再返回文件列表。音频保持常驻控制界面。
- 音频封面/歌词区域与视频画面支持左右滑动，按当前目录排序切换同类型媒体；单击视频画面显示或隐藏顶部与底部控制层。
- 视频控制层提供播放 / 暂停、前后 10 秒、可聚焦进度条、`1× / 1.25× / 1.5× / 2×` 倍速循环和显示模式循环按钮。显示模式顺序固定为：适应屏幕、铺满屏幕、拉伸全屏、原始比例、16:9、4:3。
- 适应屏幕保留完整画面，铺满屏幕和 16:9/4:3 保持比例并裁剪，拉伸全屏允许变形；原始比例按原始像素居中显示，未知尺寸时回退适应屏幕。
- 进度条获得焦点后用左右键前后跳转 10 秒；触屏可点击或水平拖动进度条跳转。所有按钮沿用 `TvButton` 焦点样式。
- 本地和 SMB 媒体均由 Media3 播放。SMB 使用 `readRange` 按播放器请求读取，播放与 seek 不完整缓存媒体文件。

### 文本预览页

- UTF-8 文本最多读取前 `1 MiB`，显示原始行号并自动换行；每页 14 个原始行。
- 上一页 / 下一页按钮可聚焦，正文获得焦点后也可用遥控器左右键或 Page Up / Page Down 翻页。
- 正文区域支持触屏左右滑动翻页，且始终保留可点击的上一页 / 下一页按钮。

### 弹窗

连接管理是独立 Activity，提供 SMB 的新增、编辑、删除。右上角仅保留“添加网络地址”，无连接时屏幕中央只显示“当前暂无网络连接”。连接列表、新增和编辑表单在低高度设备上可纵向滚动，输入项获得遥控器焦点时必须自动滚入可视区域。添加时先打开协议选择弹窗，当前仅提供 SMB，选择后再进入新增 SMB 表单。所有操作必须可被遥控器聚焦和触屏点击。网络 Dock 没有已保存连接时，在屏幕中心展示“去配置”按钮并跳转至连接管理。

```text
┌ 编辑 SMB ───────────────────────────────────────────┐
│ 修改已保存的网络连接信息。                           │
│ [名称                                             ] │
│ [主机                                ] [端口       ] │
│ [共享目录                                         ] │
│ [用户名                              ] [密码       ] │
│ [域（可选）                                       ] │
│                                  [取消]    [保存]   │
└─────────────────────────────────────────────────────┘

┌ 删除连接 ───────────────────────────────────────────┐
│ 将删除“客厅 NAS”，此操作无法撤销。                   │
│                                  [取消]    [删除]   │
└─────────────────────────────────────────────────────┘
```

连接管理列表沿用分组表面：`8dp` 圆角、`1dp` 弱描边、最小高度 `84dp`，标题、说明和操作分别使用 `22sp / 18sp / 20sp`。协议选择、新增、编辑和删除弹窗统一使用表面色、`8dp` 圆角、`1dp` 弱描边和零色调海拔；弹窗标题 `28sp`，正文 `18sp`，输入文字 `20sp`，字段间距 `12dp`。输入框使用 `8dp` 圆角，默认描边取主题轮廓色，聚焦描边、标签和光标使用次要青色；普通操作继续使用琥珀焦点态。删除为危险操作，静止态使用错误色文字和描边，聚焦态使用错误色背景、深色文字并保留统一的浅琥珀焦点描边。弹窗操作按钮固定为 `112dp` 并居中文字；主机与共享目录未填写时禁用保存。

设置是独立 Activity，左侧一级菜单栏包含显示设置、缓存设置和关于，以 `260dp` 为 `100%` 显示缩放基准，并随显示缩放在 `208dp-312dp` 间同步变化。侧栏使用设置侧栏色，右侧沿用背景色，两侧用 `1dp` 设置分隔线分开。

```text
+----------------------+--------------------------------------------------+
| 设置                 | 关于                                             |
|                      | ┌──────────────────────────────────────────────┐ |
| ● 显示设置           | | 项目信息                         v0.1.0 / ABI | |
|   缓存设置           | | 本地与 SMB / NAS 媒体文件管理器               | |
|   关于               | |----------------------------------------------| |
|                      | | 检查更新                             功能开发中 | |
| ← 返回               | |----------------------------------------------| |
|                      | | GitHub                                  打开 | |
|                      | | github.com/gbandszxc/good-tvplorer           | |
|                      | └──────────────────────────────────────────────┘ |
+----------------------+--------------------------------------------------+
```

设置页侧栏内边距为水平 `28dp`、垂直 `32dp`，菜单项间距为 `8dp`，菜单项高度为 `64dp`。当前页面使用表面色背景和 `8dp` 前导圆点共同表示，并暴露 `selected` 语义；不使用侧边短线。遥控器焦点使用主色背景、`3dp` 浅琥珀描边与深色文字，覆盖当前页面的视觉样式，但保留其选中语义，确保焦点与选中状态不会混淆。

右侧内容区使用水平 `36dp`、垂直 `32dp` 的内边距；页面标题固定，标题与可滚动内容间距为 `24dp`。缓存和关于页面各使用一个 `8dp` 圆角、`1dp` 描边、`4dp` 内边距的分组表面，内部以紧凑设置行和分隔线组织信息，不再堆叠同构大卡片。设置行最小高度为 `84dp`，内边距为水平 `20dp`、垂直 `14dp`，不限制最大高度；文字换行时行高随内容增加。

静态信息行不可聚焦、不可点击；清理缓存、检查更新和 GitHub 等操作行可聚焦、可点击，并使用一致的按钮语义与焦点反馈。当前缓存和项目信息为静态行。检查更新保留为占位操作行，说明为“功能开发中，暂不检查或下载更新”；按 OK 后不联网、不下载，停留在当前页面并短暂提示“检查更新功能开发中”。GitHub 行展示 `github.com/gbandszxc/good-tvplorer`，确认后通过系统浏览器打开项目主页，无可用浏览器时保持当前页面。

显示设置中的减号、百分比和加号使用固定宽度，百分比始终居中；减号与加号分别提供“减小字体”和“增大字体”的可访问性说明。“恢复默认”使用 `128dp` 紧凑宽度，文字居中。`80%` 时禁用减号，`120%` 时禁用加号，恢复默认回到 `100%`。设置页焦点颜色与描边动画统一使用 `160ms`，不使用弹跳或布局动画。右侧内容在低高度设备或较大字体档位下可纵向滚动，焦点进入屏幕外设置行时必须将该行完整滚入可视区域。

## 状态

- 加载：MVP 阶段可使用 spinner 加清晰文案；后续列表加载优先使用骨架行。
- 空状态：使用明确文案，例如 `目录为空`。
- 错误：展示用户可理解的消息，不泄露密码或日志细节。
- 预览加载：在黑色或深色背景中心显示 spinner。
- 原图已有缩略图时保留缩略图作为加载占位，避免黑屏；加载失败时保留占位并叠加可理解的错误文案。
- 浏览锚点：本机和每个 SMB 来源的每一级目录记住最近进入的子目录；返回或切回该目录时，自动聚焦该条目并沿用琥珀焦点态。

## 动效

动效应短促，并且只由状态变化驱动。

- 焦点过渡：颜色 / 描边变化控制在 150-200ms；不要弹跳。
- 加载 / 预览过渡：简单淡入淡出或瞬时切换。
- 不做页面加载编排动画。
- 不做装饰性的弹跳、弹性或长交错动画。

## 可访问性

- 每个可操作控件都必须可聚焦。
- 每个可操作控件都必须可触屏点击；紧凑工具栏等独立触控目标不小于 `48dp`。
- 焦点状态必须在电视观看距离下清晰可见。
- 不只依赖颜色表达状态；同时使用文字和布局状态。
- 文字和焦点控件必须保持高对比度。
- 触屏可作为兜底，但主路径是方向键。

## 已知设计债

- 当前图标是文字标记（`DIR`、`IMG`、`TXT`、`AUD`、`VID`、`FILE`）；后续替换为一致的图标体系。
- 加载态目前使用居中 spinner；浏览器体验打磨时补充列表骨架屏。
- SMB 文本输入在电视端负担较重；后续增加更好的已保存 / 最近连接流程。

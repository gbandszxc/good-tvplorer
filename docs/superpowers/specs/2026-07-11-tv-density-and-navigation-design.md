# TV 信息密度与快速导航设计

## 目标

保持现有首页、浏览器三栏和预览页布局不变，让 45 英寸以上电视在约 3 米观看距离下显示更多信息，并统一各页面的字号与空间层级。

```text
+--------------------------------------------------------------------------------+
| Good TVplorer | ✎ smb:/Movies/2024/very-long-directory… | [网格][刷新][显示][返回] |
+----------------+------------------------------------------+--------------------+
| 位置           | 文件网格 / 列表                           | 快速预览           |
| 本地           | [封面][封面][封面][封面]                 | 缩略图             |
| SMB / NAS      | [封面][封面][封面][封面]                 | 文件名、类型、大小 |
+----------------+------------------------------------------+--------------------+
```

## 缩放语义与显示设置

- 当前 `75%` 的实际文字体量成为新的 `100%` 基线。
- 存储值调整为 `0.80f..1.20f`，默认 `1.00f`，步长 `0.05f`；应用字体密度为固定基线 `0.75f × 存储值`。
- 设置弹窗提供 `−`、当前百分比和 `＋` 三个遥控器可聚焦控件；边界值禁用对应按钮。
- 增加“恢复默认”，写入 `1.00f`；“关闭”仅关闭弹窗，调整实时保存，无额外确认步骤。
- 弹窗沿用 `#101A26` 面板、`#152232` 普通按钮、琥珀焦点和现有 8dp 圆角；标题 / 说明 / 按钮分别使用既有紧凑字号层级，不引入参考图的蓝色或展示字体。

## 浏览顶部栏

- 顶部栏分为固定品牌区、可伸缩地址区和固定操作区。
- 操作区的“网格 / 刷新 / 显示 / 返回”各自预留固定最小宽度，路径再长也不可挤压或换行。
- 地址区默认展示路径并单行省略；其左侧提供项目内的编辑图标按钮。
- 编辑后地址区替换为可聚焦的单行输入框。OK / IME 完成提交；Back 取消并恢复当前地址。
- 输入只在当前文件源内解析：相对路径基于当前目录，`/` 开头从该文件源根目录开始。提交时复用现有 `openBrowser(sourceKey, path)`，由既有读取错误 UI 展示无效或不可访问路径的原因。

## 密度与一致性

- 保持已有三栏比例和焦点可见性；减少顶部栏、轨道、预览面板、网格和列表的内外间距至一致的 4dp 倍数。
- 网格最小卡片宽度收紧，增加一列可见文件；卡片缩略图仍保持 16:10。
- 列表行不低于现有遥控器可操作高度，长文件名继续单行截断。
- 统一为页面标题、主操作 / 文件标题、元信息、提示四级文字；不新增字体或主题 token。

## 验证

- 单元测试覆盖缩放基线换算、80% / 120% 边界和 5% 步进。
- 构建 `:app:assembleDebug`；手动核验长路径、地址编辑的 OK / Back、无效路径报错与顶部四按钮不收缩。
